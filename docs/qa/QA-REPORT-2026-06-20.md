# MADIS QA Report

**Tanggal:** 20 Juni 2026  
**Versi target:** 1.0.26  
**Lingkungan:** Local browser QA dan production smoke test

## Ringkasan

QA dilakukan sebelum melanjutkan roadmap improvement. Fokus pengujian mencakup:

- service worker dan pembaruan cache;
- login, JWT, proteksi halaman, dan role Admin;
- pemuatan master data Departemen;
- seluruh halaman utama;
- endpoint file/R2;
- struktur HTML, referensi aset, dan syntax JavaScript;
- ketersediaan dependency CDN.

## Bug Yang Diperbaiki

1. **Pembaruan aplikasi memerlukan hard refresh**
   - `version.json` sebelumnya dapat tersimpan dalam cache.
   - Versi tersebar dan tidak konsisten di beberapa modul.
   - Sekarang semua halaman menjalankan bootstrap pembaruan yang sama.
   - `sw.js` dan `version.json` diperiksa tanpa cache.
   - JavaScript/CSS memakai revalidation dan service worker mengambil alih otomatis.

2. **Departemen tidak muncul karena kode lama masih aktif**
   - Service worker lama dapat mempertahankan modul master data/dashboard.
   - Cache aplikasi sekarang dihapus saat versi release berubah.
   - Dropdown Departemen pada Kelola Karyawan sekarang memakai tabel master
     Departemen sebagai sumber utama, bukan hanya nilai yang kebetulan tersimpan
     di record karyawan.

3. **Markup `<body>` rusak pada 11 halaman**
   - Ditemukan bentuk `<body<!-- Sidebar -->` dan `<body` tanpa penutup.
   - Seluruh halaman telah dinormalisasi menjadi HTML yang konsisten.

4. **Jakarta Cardiovascular tidak redirect saat sesi habis**
   - Halaman dapat tertahan pada loading overlay.
   - Sekarang redirect ke login dan logout benar-benar membersihkan sesi.

5. **Redirect MCU Expiry salah**
   - Redirect tanpa sesi menuju path yang salah.
   - Sekarang menuju `pages/login.html`.

## Batasan QA

- Mutasi data medis production tidak dilakukan.
- Upload/download/delete file dengan akun production belum diuji end-to-end,
  karena memerlukan sesi pengguna dan file uji yang disetujui.
- MCU Expiry hanya bisa diuji penuh di production karena servicenya memang
  bergantung langsung pada Supabase.
- Migrasi `migrations/secure-rls-authenticated.sql` masih perlu dipastikan sudah
  dijalankan pada Supabase agar proteksi database lengkap.
